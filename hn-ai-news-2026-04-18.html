Are the costs of AI agents also rising exponentially? (2025) (tobyord.com)
Toby Ord charts exponential cost curves for AI agents — and what it means for human labor substitution
302 points  |  Apr 17, 2026 UTC  |  129 comments
key points & discussion
  • Toby Ord analyzes the hourly cost of capable AI agents over time, finding that costs are falling exponentially — potentially faster than performance is improving.
  • The analysis frames the question of AI agent economics similarly to cloud vs. on-premise compute decisions: at what cost crossover does it make sense to use AI vs. hire a human?
  • Uses METR task-completion benchmark data as the capability axis, plotting against API cost per hour of agent runtime.
Discussion: HN thread
  • Commenters noted that using METR data obscures the picture — comparing against open-model cost frontiers (e.g. GLM-5.1, MiniMax-M2.7) paints a very different cost trajectory.
  • The cloud vs. on-premise analogy resonated: some predicted software engineering project estimates will shift from time-based to capex/token-budget-based within a few years.
  • Skeptics argued the framing assumes agent capabilities are fungible with human work hours — a substitution that hasn't materialized cleanly in practice.

Average is all you need (rawquery.dev)
The danger of accepting AI-generated SQL without verification — average outputs cause real data errors
112 points  |  Apr 17, 2026 UTC  |  100 comments
key points & discussion
  • The post argues that AI-generated SQL looks correct but often produces plausible-but-wrong results — especially for attribution, aggregation, and edge-case logic.
  • The core problem: users who can't write SQL also can't verify AI-generated SQL, creating a blind trust loop.
  • Title is a riff on "Attention is all you need" — arguing that "average" (median, plausible) output is what transformers naturally produce.
Discussion: HN thread
  • Top comment: "you asked a question in English and got a table — but nobody checked if it was correct." The follow-up response noted no one said you didn't have to check either.
  • Key rebuttal: if you don't know SQL and your only view into the data is SQL, how do you verify the output? The tool creates a dependency it can't resolve.
  • Several commenters pointed out the problem is not AI-specific — people have always run Excel formulas or BI queries without understanding them; AI just lowers the barrier to doing so.
