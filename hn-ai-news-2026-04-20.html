Zero-Copy GPU Inference from WebAssembly on Apple Silicon (abacusnoir.com)
Technique for running ML inference in the browser with zero CPU→GPU memory copies on Apple Silicon via WebGPU
112 points  |  Apr 18, 22:46 UTC  |  51 comments
key points & discussion
  • Exploits Apple's Unified Memory Architecture — CPU and GPU share physical memory — to avoid costly data copies during WebAssembly inference.
  • Uses WebGPU's buffer mapping API to expose GPU-accessible pointers directly to WASM without intermediate copies.
  • Benchmarks show meaningful latency improvements for models running in Safari / Chrome on M-series Macs.
Discussion: HN thread
  • Hardware veterans pushed back on the "Apple Silicon uniquely enables this" framing — x86 iGPUs and dGPUs have had dynamic shared-memory APIs for over a decade.
  • The real novelty is exposing this capability through the browser sandbox (WebGPU + WASM), not the underlying UMA architecture itself.
  • Discussion of why discrete GPUs still bring dedicated VRAM: lower-latency, higher-bandwidth memory that DDR can't match even without a physical copy.
