Zero-Copy GPU Inference from WebAssembly on Apple Silicon (abacusnoir.com)
How Apple Silicon's unified memory enables zero-copy GPU inference from WASM — and why x86 can't
113 points  |  Apr 19, 2026 UTC  |  51 comments
key points & discussion
  • On Apple Silicon, CPU and GPU share physical memory — ML tensors need no copy operation to move between processor types, eliminating a key inference latency bottleneck.
  • WebAssembly inference runtimes can directly exploit this on Apple Silicon, reducing both latency and memory overhead significantly vs. traditional GPU inference pipelines.
  • The advantage is architectural and doesn't apply to x86 with discrete GPUs, which have separate VRAM requiring explicit DMA transfers.
Discussion: HN thread
  • x86 iGPU RAM reservation clarified: that fixed boot-time allocation is for the framebuffer, not true unified memory — architecturally different from Apple's implementation.
  • Discrete GPU VRAM exists because it's physically different memory optimised for GPU bandwidth — the latency to copy from host RAM to VRAM is the bottleneck that UMA eliminates.
  • Some pushback on Apple marketing framing: UMA is real and beneficial, but it's not as novel as Apple's presentation suggests — other vendors have had partial implementations for years.
