        time: "May 23, 12:04 UTC"
        content_bullets:
          - "Three bottlenecks govern deep learning speed: compute (FLOPs), memory bandwidth (data movement), and overhead (Python/framework dispatch) — and most ops are bandwidth-bound, not compute-bound."
          - "An A100 GPU does 312 TFLOPS for matmul but only 1.5 TB/s memory bandwidth; in the time Python executes one FLOP, the GPU could have done 9.75 million — making CPU-side overhead a serious drag."
          - "Operator fusion is the most impactful fix: merging two ops into one kernel halves memory accesses, ~2x speedup by eliminating intermediate round-trips to global GPU memory."
          - "The roofline model gives a principled diagnosis — measure achieved FLOPS vs. peak to determine if you're compute-bound, bandwidth-bound, or overhead-limited before choosing an optimization."
          - "Activation checkpointing (rematerialization) can paradoxically reduce both memory and runtime: recomputing intermediates is cheaper than the bandwidth cost of storing and reloading them."
        discussion_bullets:
          - "Commenters note the A100 FLOP gap is partly single-core vs. massively parallel hardware, but the deeper point — that Python overhead and memory latency dominate small ops — remains valid."
          - "A thread on fused operations clarifies the speedup comes from avoiding slow global memory round-trips between kernel launches, not anything special about the specific operations fused."
          - "Several readers highlight Karpathy's llama2.c (~300 lines of C) as a practical entry point for understanding transformer inference before diving into GPU optimization."
