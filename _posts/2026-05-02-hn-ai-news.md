    content_bullets:
    - 'SKILL.md files work as three-tier loader specs: frontmatter (~100 tokens, always scanned for routing), body (triggered
      load, max 500 lines), and referenced files/scripts (on-demand, unlimited capacity).'
    - Splitting a 1,200-line monolithic skill into a 180-line spine with three references cut context usage from 20% to 7%
      of the context window with identical outputs.
    - 'Key antipatterns: adding frontmatter to non-skill reference files (promotes them to routing level), hardcoding file
      paths instead of letting agents discover them, and omitting a ''Gotchas'' section for environment-specific quirks.'
    - Skills are calibrated to a specific model's compliance behavior — upgrading from Sonnet to Opus can break a skill because
      more capable models interpret instructions literally rather than applying judgment.
    - 'The architectural rule: metadata routes, the skill body instructs, scripts inject results (not source) — treating these
      tiers interchangeably is the root cause of bloated, fragile agents.'
    discussion_bullets:

    time: May 1, 12:23 UTC
    content_bullets:
    - AutoRound is Intel's open-source toolkit that compresses LLMs (large AI models) down to 2–4 bits using sign-gradient
      descent optimization, enabling models to run on far less memory.
    - A ~200GB DeepSeek-R1 model quantized to INT2-mixed retains 97.9% accuracy; at 4-bit the toolkit claims leading benchmark
      results vs. other quantization methods.
    - Quantizing a 7B-parameter model takes ~10 minutes on a single GPU; an 'auto-round-light' recipe offers 2–3x faster turnaround
      with modest accuracy trade-offs.
    - AutoScheme automatically generates adaptive mixed-precision recipes (assigning different bit widths per layer) in minutes,
      reducing manual tuning.
    - Outputs export to multiple formats (GGUF, AutoGPTQ, AutoAWQ, vLLM) and runs on CUDA GPUs, Intel XPU, Gaudi HPU, and
      Xeon CPUs.
    discussion_bullets:
    - AutoRound claims to push Q4 quantization accuracy from ~99–99.8% of BF16 up to ~99.4–100%; skeptics note near-perfect
      claims are common in quantization papers, and compression still degrades performance on long contexts.
    - One commenter shared a live demo running a Qwen 35B model quantized with AutoRound at just 11.65 GB with 300k context,
      illustrating practical deployment gains.
    - Discussion touched on Quantization-Aware Training (QAT) as an alternative for extreme 2–3 bit compression, but the consensus
      was that QAT has lost momentum because inference hardware is far cheaper than the compute needed to QAT a full FP16
      model.
