    points: 37
    hn_url: https://news.ycombinator.com/item?id=48667409
    comments: 17
    time: Jun 25, 02:27 UTC
    content_bullets:
    - Using native Web APIs (URL, FormData, AbortSignal, Promise.allSettled) instead of custom implementations slashes output tokens by 7-10x on complete request handlers.
    - 'Individual gains are large: URLSearchParams cuts query parsing from ~140 to ~12 tokens; native FormData drops multi-field forms from 200+ to ~14 tokens.'
    - Code formatting reduces input tokens by ~24.5%, but since input costs only 20-33% of output costs, cutting output via native APIs is the bigger lever.
    - 'Comments matter: models treat them as authoritative direction, so stale comments degrade output while design-intent comments improve it.'
    - The practical fix is explicit prompting — tell the LLM which native APIs to use and which custom patterns to avoid, rather than relying on general style guidance.
    discussion_bullets:
    - Commenters note that verbose, descriptive variable names and inline comments increase input token counts but improve LLM output quality — a genuine tension between cost and clarity.
    - Enforcing a consistent coding style via linting helps LLMs reproduce that style reliably; inconsistent style causes models to average across training patterns instead of anchoring on your codebase.
    - With context-sensitive pricing on newer models, token profiling work like this has direct monetary value — small reductions in average context size compound quickly at scale across thousands of API
      calls.
