  - title: Universal Memory Protocol – a shared format for agent memory
    link: https://universalmemoryprotocol.io/
    domain: universalmemoryprotocol.io
    summary: Universal Memory Protocol proposes a vendor-neutral standard for persistent
      AI agent memory, filling the gap between MCP (tool calls) and A2A (agent communication)
    points: 39
    hn_url: https://news.ycombinator.com/item?id=48428796
    comments: 32
    time: Jun 6, 20:49 UTC
    content_bullets:
    - UMP defines six core operations (capabilities, recall, remember, revise, forget,
      get) so any conforming client can be implemented in ~100 lines of code.
    - Records are typed, scoped, bi-temporal, and DID-signed JSON — reusing W3C PROV
      standards; facts are closed and linked to successors rather than overwritten.
    - Memory categories follow episodic (events), semantic (facts), and procedural
      (how-to) patterns, stored across JSON files, SQL, Redis, or vector engines.
    - 'Security is a first-class concern: records are content-addressed, DID-signed,
      and the spec mandates filtering pipelines since memory is treated as attacker-controllable
      input.'
    - Four conformance tiers (L0–L3) allow incremental adoption from simple JSON exports
      up to full runtime memory-sharing across agents and vendors.
    discussion_bullets:
    - Commenters frame UMP as 'MCP for persistent memory' — the missing third interoperability
      layer after tool calls and agent-to-agent communication.
    - Skeptics question whether another memory standard is needed; proponents argue
      the goal is interoperability between frameworks (likened to RSS), not replacing
      existing stores.
    - 'Privacy observers note the dual nature of standardized agent memory: powerful
      for continuity, but concerning as a centralized record of everything agents
      learn about users.'
